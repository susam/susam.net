If you are a beginning Emacs user, or maybe even an experienced one, but have never used Emacs keyboard macros earlier, learning this one feature will change the way you use Emacs forever. I know because I was a beginner once. As a beginner, I often used to rely on writing clever regular-expressions-based search-and-replace commands to perform complex editing tasks.

Once I learnt keyboard macros, it changed the way I performed those routine and repetitive editing tasks. I would no longer resort to writing regular expressions to solve most editing problems. Keyboard macros let you solve problems the dumb way. Just carry on with your editing tasks and ask Emacs to remember and replay them when you want to repeat them. The following famous quote about regular expressions is quite relevant in this context:

Some people, when confronted with a problem, think "I know, I'll use regular expressions." Now they have two problems.

Keyboard macros avoid the second problem. They let you focus on your original problem: the complex editing task you want to perform.

While learning keyboard macros can change the way you use Emacs, there's more! Learning how to name keyboard macros is going to change the way you use Emacs for a second time. Once you know how to name macros, you can invoke them like ordinary Emacs commands with M-x. Yes, you can start creating your own Emacs commands without having to write a single line of Emacs Lisp (Elisp) yourself! Emacs will generate the Elisp code necessary to define the new commands for your keyboard macros.

In this post, we will take a look at these features. We will start with the basics of keyboard macros, how to define them, how to execute them, how to edit them, etc. Then we will see how to name them, create key bindings for them, how to create Emacs commands out of them. All of this information is available in the Emacs manual too (see section Keyboard Macros) but I am hoping that this post and especially the demos presented here will make it more comfortable to learn about and start using keyboard macros in Emacs.

The sections below are arranged in such a manner that if you decide to stop at some section and step away, you would have still learnt some useful techniques that you can adopt in your day-to-day Emacs life. You can always come back later to read the other sections.

Record and Replay

From here on, some familiarity with Emacs is expected. For example, it is assumed that you know that the key sequence notation C-x e means typing ctrl+x followed by e. Similarly, the key sequence M-x means typing alt+x or option+x depending on the type of keyboard you have. The key RET represents the return or enter key. The notation SPC represents a space character. We will use the notation <f3> and <f4> to represent the function keys F3 and F4, respectively.

Let us record a keyboard macro. We will perform a very simple task of adding a prefix and a suffix to the current line. The prefix is going to be [[[[. The suffix is going to be ]]]]. This editing example is intentionally too trivial and silly, so that we can focus on how to work with keyboard macros rather than getting lost in the details of a particular editing task. Let us assume the buffer initially looks like this and the cursor is at the first line: 

Emacs is the advanced, extensible,
customizable, self-documenting editor.
GNU Emacs is free software. Everyone is
free to use it and free to redistribute
it under certain conditions.

The block shows where the cursor is. Now perform the following steps:

  1. Type <f3> to start recording a keyboard macro. Note how the modeline displays Def in the list of minor modes. This is an indicator that the minor mode named defining-kbd-macro is now active. Any subsequent keyboard input is going to be recorded as a keyboard macro.

  2. Type C-a to go to the beginning of the line. Do this even if the cursor is already at the beginning of the line. Doing this ensures that when we replay the keyboard macro, the cursor always moves to the beginning of the line, so we can later replay the keyboard macro on any line irrespective of where the cursor is on that line.

  3. Type [[[[ to insert our chosen prefix.

  4. Type C-e to go to the end of the line.

  5. Type ]]]] to insert our chosen suffix.

  6. Type C-n to move to the cursor to the next line. This may seem like an odd thing to do but the usefulness of this will become apparent when we replay the keyboard macro multiple times. After executing the keyboard macro once, the cursor will automatically move to the next line so that merely repeating the keyboard macro call would repeat the same operation on the next line. This step ensures that the cursor is left at a good place when the keyboard macro call ends, so that it can be repeated conveniently.

  7. Type <f4> to complete recording the keyboard macro. You could also type C-x ) instead. The buffer should look like this now: 

    [[[[Emacs is the advanced, extensible,]]]]
customizable, self-documenting editor. 
GNU Emacs is free software. Everyone is
free to use it and free to redistribute
it under certain conditions.

  8. The keyboard macro is now ready. Type <f4> once to replay the keyboard macro. The buffer looks like this now: 

    [[[[Emacs is the advanced, extensible,]]]]
[[[[customizable, self-documenting editor.]]]]
GNU Emacs is free software. Everyone is 
free to use it and free to redistribute
it under certain conditions.

  9. Notice how the cursor has moved to the next line automatically. This is a result of recording C-n too as part of the keyboard macro in step 6 earlier. As a result, we can simply repeat this keyboard macro by typing <f4> over and over again. Type <f4> twice now and the buffer looks like this: 

    10. [[[[Emacs is the advanced, extensible,]]]]
[[[[customizable, self-documenting editor.]]]]
[[[[GNU Emacs is free software. Everyone is]]]]
[[[[free to use it and free to redistribute]]]]
it under certain conditions.

Note that instead of using <f3> and <f4> we can also use C-x ( and C-x ), respectively, to start and stop recording a keyboard macro. This can be useful if you want to avoid function keys which can be clumsy to type on some keyboards. Similarly, instead of typing <f4> to replay a keyboard macro, we can type C-x e instead.

We could have solved this simple editing task with a regular-expression based search-and-replace operation using the key sequence C-M-% .* RET [[[[\&]]]] RET. However, for more involved editing tasks where writing a regular expression may be unwieldy, keyboard macros may turn out to be very useful.

Moving Back and Forth

Soon after you adopt recording and playing keyboard macros in your day-to-day editing tasks, you will soon realize that <f4> or C-x e replays the last keyboard macro. But sometimes you may want to replay an older keyboard macro. The keyboard macros that we record goes into a data structure known as the keyboard macro ring. We can go back and forth in this ring using the following key sequences:

  • C-x C-k C-p: Go back to the previous keyboard macro.
  • C-x C-k C-n: Go forward to the next keyboard macro.

Every time we navigate the keyboard macro ring in this manner, the selected keyboard macro is displayed in the echo area at the bottom of the Emacs frame. Once we are happy with the keyboard macro we have selected, we can type <f4> or C-x e and the selected keyboard macro is replayed.

Note that in the remaining sections below, whenever we use the term last keyboard macro, what it really refers to is the current keyboard macro in the macro ring which is often the last keyboard macro we recorded but if we have navigated to a different keyboard macro in the macro ring, then the term last keyboard macro refers to that keyboard macro.

Repeating Keyboard Macros

We can repeat keyboard macros like we repeat most other commands in Emacs: by using numeric prefix arguments. For example, typing C-3 <f4> replays a keyboard macro three times. So does C-3 C-x e.

If we want to repeat the keyboard macro for the remainder of the entire buffer, we could use the zero prefix argument, e.g., C-0 <f4> or C-0 C-x e. Technically, it repeats the keyboard macro until there is an error. As a result, it repeats the keyboard macro until it is repeated on the last line of the buffer and the cursor cannot move down any further.

Naming Keyboard Macros

Now that we have seen how to perform basic operations with keyboard macros, let us see how to name the macros. This is what I earlier claimed would change the way you Emacs for a second time! Here are the steps:

  1. If you haven't already, perform steps 1 to 7 from the Record and Replay section to record a keyboard macro.

  2. Type C-x C-k n decorate RET to name the last keyboard macro as decorate.

  3. Type M-x decorate RET to replay the keyboard macro.

Did you see what happened here? We assigned a name to our keyboard macro and it immediately became available as an Emacs command that we can now call from Elisp or execute directly using the M-x key sequence.

Saving Keyboard Macros

After we have named a keyboard macro as explained in the previous section, we can save them too in our initialization file. Assuming the initialization file is present at ~/.emacs.d/init.el, perform these steps to save the last keyboard macro:

  1. Type C-x C-f ~/.emacs.d/init.el RET to open the initialization file.

  2. Now move the cursor to a place in this file where Emacs should write the keyboard macro definition too.

  3. Type M-x insert-kbd-macro decorate RET. Emacs generates the Elisp code for the keyboard macro named decorate and inserst it as a command into the file. The generated code may look like this:

  4. Now save the file with C-x C-s. Then restart Emacs and type M-x decorate RET to confirm that the command decorate indeed replays our keyboard macro.

The code generated in step 3 may look like this: 

(fset 'decorate
   (kmacro-lambda-form [?\C-a ?\[ ?\[ ?\[ ?\[ ?\C-e ?\] ?\] ?\] ?\] ?\C-n] 0 "%d"))

To summarize, not only Emacs turns any named keyboard macro into a command but it can also generate the Elisp code for it, so that we can write it to our initialization file and save our keyboard macros. I believe this is one of the most user friendly features of Emacs. It lets you create new commands without having to write a line of Elisp code yourself!

Many users happen to collect a large collection of macros in this manner for repetitive editing tasks. It is advisable to name such macros with a common prefix, e.g., mac-, kb-, your initials, or some such pattern, so that they are easy to locate while invoking them via M-x followed by TAB-based completion.

Key Bindings for Keyboard Macros

The previous two sections explain how we can name a macro and how we can store named macros persistently in our initialization file. Now, we will see how to create key bindings for our macros. The key sequence to create a key binding is C-x C-k b. This command prompts for a key sequence which is then bound to the last keyboard macro. For example, typing C-x C-k b C-c a binds a keyboard macro to the key sequence C-c a.

If the long key sequence necessary to create a key binding for a macro bothers you, the C-x C-k b command offers a convenient shortcut. After typing C-x C-k b, if we type any digit or an uppercase letter, the macro is bound to C-x C-k char where char represents the digit or uppercase letter we typed. For example, C-x C-k b 0 binds a keyboard macro to C-x C-k 0. Similarly, C-x C-k b A binds a keyboard macro to C-x C-k A.

This can be a convenient way of defining temporary key bindings for frequently used keyboard macros in an edition session.

Editing Keyboard Macros

Another useful feature among the various keyboard macro features is the ability to edit an existing macro. Let us assume we have recorded the keyboard macro introduced in the section Record and Replay. But then we realize that we want the suffix to be }}} instead of ]]]. We can do so by performing the following steps:

  1. If you haven't already, perform steps 1 to 7 from the Record and Replay section to record a keyboard macro.

  2. Type C-x C-k e to edit the keyboard macro. The following prompt appears: 

    Keyboard macro to edit (C-x e, M-x, C-h l, or keys):

  3. Type C-x e to edit the last keyboard macro.

  4. A new buffer named *Edit Macro* with the following content appears: 

    ;; Keyboard Macro Editor.  Press C-c C-c to finish; press C-x k RET to cancel.
;; Original keys: C-a [[[ C-e ]]] C-n

Command: last-kbd-macro
Key: none

Macro:

C-a			;; move-beginning-of-line
[[[			;; self-insert-command * 3
C-e			;; move-end-of-line
]]]			;; self-insert-command * 3
C-n			;; next-line

    This is the keyboard macro editor buffer. We can edit this buffer freely to modify the keyboard macro definition. Move the cursor to the last but one line and edit that line to replace ]]] with }}}.

  5. Observe the top line of the buffer to see the editing key sequences supported. Type C-c C-c to complete editing the macro. The keyboard macro editor buffer disappears.

  6. Now go to any line of text in any buffer and type <f4> or C-x e to replay the edited macro.

Notice that in step 1, when we type C-x C-k e, we are presented with a prompt that expects an input. The possible inputs are listed out in the prompt. In the steps above, we type C-x e to select the option that edits the last keyboard macro. Let us take a look at all the available options now:

  • C-x e: Edit the last keyboard macro. For example, type C-x C-k e followed by C-x e to invoke this. This is what we did above.

  • M-x: Edit a named macro. For example, type C-x C-k e followed by M-x decorate RET to edit a keyboard macro named decorate.

  • C-h l: Edit lossage to create a keyboard macro. For example, type C-x C-k e followed by C-h l to edit lossage and create a keyboard macro. The term lossage is just a fancy term for a certain number of last input keystrokes. By default, the lossage contains the last 300 keystrokes. When we edit lossage in the keyboard macro editor, we can go through our history of keystrokes, edit it freely, keep the keystrokes we want, remove everything else, and type C-c C-c to create a new keyboard macro.

  • keys: Yes, there is an option that merely says keys. This means that we can simply type a key sequence some keyboard macro is bound to in order to edit that keyboard macro. For example, type C-x C-k e followed by C-c a to edit a keyboard macro bound to the key sequence C-c a.

There is a nice pattern in the key sequences above. In all except one case, to edit a keyboard macro, we type C-x C-k e followed by the key sequence that invokes the keyboard macro. So that makes the key sequences easy to remember. The only key sequence that does not quite fit this pattern is the one involving C-h l. We type C-x C-k e followed by C-h l to edit the lossage in the keyboard macro editor. This is not hard to remember either because normally, the key sequence C-h l invokes the view-lossage command which shows the lossage in a read-only buffer, so it seems apt that C-x C-k e followed by C-h l edits the lossage in the keyboard macro editor.

Some of the keyboard macro editing tasks mentioned in the list above can be performed with the following simpler key sequences available in the global keymap:

  • C-x C-k C-e: Edit the last keyboard macro.
  • C-x C-k l: Edit lossage to create a keyboard macro.
